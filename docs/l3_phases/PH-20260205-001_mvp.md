# PH-20260205-001: MVP

## 目的

鉱石の採取レートを入力し、作成可能なアイテムをインタラクティブに選択して生産計画を出力できる最小限のツールを構築する。

## 機能一覧

### F-20260205-001: 開発環境構築

**対応REQ**: -（基盤）

Vite + React + TypeScript のプロジェクトを初期化し、開発に必要なツールチェーンを整備する。

**受け入れ条件**:
- [x] `pnpm create vite` で React + TypeScript プロジェクトが作成されている
- [x] `.nvmrc` に Node.js 20.x が指定されている
- [x] Tailwind CSS が設定され、スタイルが適用できる
- [x] Zustand がインストールされている
- [x] ESLint + Prettier が設定されている
- [x] shadcn/ui が初期化されている（`components.json` 存在）
- [x] lucide-react がインストールされている
- [x] `.vscode/extensions.json` に推奨拡張機能が設定されている
- [x] `pnpm dev` でローカル開発サーバーが起動する

---

### F-20260205-002: ゲームデータ統合

**対応REQ**: -（基盤）

endfield-calc リポジトリからレシピ・アイテム・設備データを流用し、本プロジェクトで利用可能な形に統合する。

**受け入れ条件**:
- [x] `src/data/` にレシピ・アイテム・設備データが配置されている
- [x] `src/types/` に必要な型定義が配置されている
- [x] TypeScript でデータをインポートしてエラーなくコンパイルできる
- [x] README または適切な場所に MIT ライセンス表記がある
- [x] `public/images/` にアイテム・設備のアイコン画像が配置されている
- [x] `src/locales/ja/` に日本語翻訳データが配置されている
- [x] i18next が設定され、アイテム名・設備名を日本語で表示できる

---

### F-20260205-003: 鉱石レート入力

**対応REQ**: REQ-20260205-001

4種類の鉱石（源石鉱、紫晶鉱、青鉄鉱、銅鉱）の採取レート（個/分）を入力できるUIを提供する。
※銅鉱は現在ゲームデータに未実装のため、利用可能な3種（源石鉱物、紫晶鉱物、青鉄鉱物）で実装。

**受け入れ条件**:
- [x] 利用可能な鉱石それぞれに数値入力欄がある（現在3種、銅鉱追加時に拡張可能）
- [x] 入力値が Zustand store に保存される
- [x] 数値以外の入力が制限されている（または無視される）

---

### F-20260205-004: 作成可能アイテム算出

**対応REQ**: REQ-20260205-002

入力された鉱石量から、作成可能なアイテム一覧と各アイテムの最大生産量（個/分）を算出・表示する。

**受け入れ条件**:
- [x] 鉱石レート入力後、作成可能なアイテム一覧が表示される
- [x] 各アイテムに最大生産量（個/分）が表示される
- [x] 鉱石レートを変更すると一覧が再計算される

---

### F-20260205-005: インタラクティブ選択・再計算

**対応REQ**: REQ-20260205-003

ユーザーがアイテムと生産量を選択すると、残りリソースで作れるものを再計算し、選択可能なアイテムが動的に更新される。

**受け入れ条件**:
- [x] アイテムを選択して生産量を指定できる
- [x] 選択後、消費される鉱石量が計算される
- [x] 残りリソースで作れるアイテムの最大生産量が再計算される
- [x] 選択を解除すると、リソースが戻り再計算される

---

### F-20260205-006: 生産計画出力

**対応REQ**: REQ-20260205-004

選択した生産リスト（アイテム名 × 生産量/分）と必要な設備台数を表示する。

**受け入れ条件**:
- [x] 選択したアイテムと生産量の一覧が表示される
- [x] 各アイテムに必要な設備台数が表示される
- [x] 設備ごとの合計台数が表示される
